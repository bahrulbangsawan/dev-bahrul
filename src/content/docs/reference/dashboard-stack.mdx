A comprehensive guide to building a modern, performant dashboard application using cutting-edge technologies optimized for edge computing and real-time data visualization.

## Tech Stack Summary

### Frontend
- **Framework:** React 19 + TypeScript
- **Build Tool:** Bun
- **Routing:** TanStack Router
- **UI Library:** shadcn/ui (Tailwind CSS + Radix UI + Lucide Icons)
- **State Management:** Zustand
- **Data Fetching:** TanStack Query (React Query)
- **Forms:** React Hook Form + Zod
- **Tables:** TanStack Table
- **Charts:** Recharts + Tremor

### Backend
- **API Framework:** Hono (Cloudflare Workers)
- **Database:** Drizzle ORM + Neon PostgreSQL
- **Storage:** Cloudflare R2 (Object Storage)
- **Analytics:** ClickHouse
- **Real-time:** WebSocket (Durable Objects)
- **Authentication:** Better Auth

### DevOps & Tools
- **Deployment:** Cloudflare Workers + Cloudflare Pages
- **Monitoring:** Sentry Edge SDK
- **CI/CD:** GitHub Actions
- **Linting:** ESLint + Prettier
- **Testing:** Bun + React Testing Library

---

## Overview

This tech stack is designed for building **high-performance, real-time dashboards** deployed on the edge with:
- **Serverless-first architecture** (Cloudflare Workers)
- **Real-time data capabilities** (WebSocket)
- **Analytics processing** (ClickHouse)
- **Type-safe, modern DX** (TypeScript, Bun)

### Data Layer: Drizzle ORM + Neon PostgreSQL

### Cost Optimization Tips

<Aside type="tip">

**1. Maximize Free Tiers**
- Start with Cloudflare Workers (100k req/day) + Pages (unlimited)
- Use Neon free tier (10 GB) for initial development
- Leverage Sentry free tier (5k errors) for monitoring

**2. Strategic Upgrades**
- Upgrade database first when hitting storage limits
- Add ClickHouse only when you need analytics at scale
- Keep edge computing on free tier as long as possible

**3. Cost-Saving Architecture**
- Cache aggressively at the edge (Cloudflare KV/R2)
- Use Neon branching for dev/staging (no extra cost)
- Implement efficient database indexes to reduce query time
- Compress and optimize assets before deployment

**4. Monitor Usage**
- Set up billing alerts on all platforms
- Track daily request patterns
- Identify and optimize expensive queries
- Use connection pooling to reduce DB costs

</Aside>
